# 1.packmol_bulk_box

Simple PACKMOL + ASE example for building a bulk molecular box.

This example packs multiple molecular species into a fully open periodic box using PACKMOL. It is intended as a minimal starter workflow for bulk liquid or electrolyte box generation.

---

## What this example does

This workflow:
- reads a unified YAML config from `config/pack.yaml`
- uses an empty box definition (`input_box/empty_box.cif`) for box metadata
- packs all requested species into the user-defined Cartesian region
- writes a PACKMOL input file into `work/`
- runs PACKMOL
- writes the final packed structure as XYZ
- optionally converts the XYZ result to CIF using ASE

In this bulk example, the empty box file is **not** included as a fixed PACKMOL structure. It is kept for compatibility and cell-length metadata.

## Requirements

```bash
cd ../0.create_conda_env
bash install_packmol_ase.sh
conda activate packmol_ase
```

---

## Configuration

The workflow reads its settings from: `config/pack.yaml`

**Main sections:**
- `mode`: selects bulk
- `packmol`: PACKMOL command, tolerance, and random seed
- `input`: bulk box metadata file and whether to include a fixed slab stanza
- `cell`: periodic box lengths
- `pack_region`: Cartesian region used for molecule placement
- `species`: list of molecules/ions to pack
- `output`: output file stem and format options

---

## Input files

### `input_box/empty_box.cif`
This file defines the box cell lengths used by the workflow.
For this bulk example, it is kept as a simple placeholder structure with the desired box dimensions.

### `input_files/*.xyz`
Each species is provided as a separate XYZ file.
PACKMOL uses these coordinates as the molecular templates to place inside the target box.

---

## Usage

Run the example from this directory:

```bash
bash run.sh
```

The runner uses:
- `config/pack.yaml`
- `scripts/build_packed_system.py`

No command-line arguments are required for this minimal example.

---

## Output files

After a successful run, the workflow creates:

```text
work/
├─ bulk.xyz
├─ bulk.cif
├─ packmol.inp
└─ logs/
   └─ packmol.log
```

- `bulk.xyz`: Raw packed structure generated by PACKMOL.
- `bulk.cif`: CIF version of the packed structure written by ASE using the configured cell lengths.
- `packmol.inp`: Generated PACKMOL input file used for the run.
- `logs/packmol.log`: PACKMOL standard output log.

---

## Example system in this folder

This example packs:
- 16 Cl ions
- 16 Na ions
- 851 water molecules

into a:
- 30.0 × 30.0 × 30.0 Å box

---

## Notes

- The packing region is currently set to fill the full box.
- The output XYZ header is normalized after the PACKMOL run for cleaner formatting.
- This example is intentionally simple and is meant to serve as a template for larger bulk-box workflows.
