# 2.packmol_slab

Simple PACKMOL + ASE example for packing molecules into the gap region between fixed slab layers.

This example is intended for confined packing workflows, such as water/electrolyte placement between graphene sheets, electrodes, or other solid slabs.

---

## What this example does

This workflow:
- reads a unified YAML config from `config/pack.yaml`
- reads the slab structure from `input_slab/`
- uses the slab as a fixed structure in the PACKMOL input
- packs all requested molecular species only inside the user-defined gap region
- writes a PACKMOL input file into `work/`
- runs PACKMOL
- writes the final packed structure as XYZ

In this slab example, the slab is included as a fixed structure so that PACKMOL places molecules around it rather than ignoring the slab atoms.

## Requirements

This example expects:

cd ../0.create_conda_env
bash install_packmol_ase.sh

---

## Configuration

The workflow reads its settings from: `config/pack.yaml`

**Main sections:**
- `mode`: selects `slab`
- `packmol`: PACKMOL command, tolerance, and random seed
- `input`: slab structure file and whether to include the fixed slab stanza
- `pack_region`: Cartesian region used for molecule placement
- `species`: list of molecules/ions to pack
- `output`: output file stem and format options

---

## Input files

### `input_slab/graphene_slab_gap_6.0nm.xyz`
This file contains the fixed slab structure used during packing.
For this example, the slab is included in the PACKMOL input as a fixed structure so that molecules are packed only in the open gap region and do not overlap the slab atoms.

### `input_files/*.xyz`
Each species is provided as a separate XYZ file.
PACKMOL uses these coordinates as the molecular templates to place inside the target gap region.

---

## Usage

Run the example from this directory:

```bash
bash run.sh
```

The runner uses:
- `config/pack.yaml`
- `scripts/build_packed_system.py`

No command-line arguments are required for this minimal example.

---

## Output files

After a successful run, the workflow creates:

```text
work/
├─ graphene_slab_water_pack.xyz
├─ packmol.inp
├─ fixed_slab_for_packmol.xyz
└─ logs/
   └─ packmol.log
```

- `graphene_slab_water_pack.xyz`: Final packed structure generated by PACKMOL.
- `packmol.inp`: Generated PACKMOL input file used for the run.
- `fixed_slab_for_packmol.xyz`: Temporary XYZ copy used internally for the fixed slab structure in the PACKMOL input. This file is mainly a helper file for the workflow. It is not the main result.
- `logs/packmol.log`: PACKMOL standard output log.

## Example system in this folder

This example packs:
- 8 Cl ions
- 8 Na ions
- 374 water molecules

inside the region:
- x = 0.618 to 11.732 Å
- y = 0.356 to 12.477 Å
- z = 2.0 to 58.0 Å

The slab structure is taken from: `input_slab/graphene_slab_gap_6.0nm.xyz`

---

## Notes

- The packing region is explicitly defined in `config/pack.yaml`.
- In slab mode, `include_slab_stanza: true` means that the slab is inserted into the PACKMOL input as a fixed structure.
- CIF output is currently disabled in this example. If you later want CIF output, you should provide valid cell lengths in the config.
